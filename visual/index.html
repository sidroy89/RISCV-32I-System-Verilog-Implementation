    What is this?

    An interactive, cycle-accurate visualizer for a RV32I single-cycle CPU built in SystemVerilog. Write RISC-V assembly, assemble it, then step through each cycle to watch data and control signals flow through every wire in the datapath in real time.

    How to Use
    1Write assembly in the editor or pick an example from the dropdown.
    2Click Assemble & Load to compile and reset the CPU.
    3Use ▶| to step one cycle, ▶ to play continuously, ◀◀ to step back.
    4Toggle HEX in the toolbar for readable addresses (PC starts at 0x80000000 — see below).
    5Pan by dragging the canvas, zoom with scroll or the +/− buttons.
    Wire Color Legend
    Data wire
    32-bit values — PC, register data, ALU result, memory read data
    Control wire (dashed)
    1-bit signals from the Decoder — mux selects, write-enables, ALU op
    Instruction bus
    Raw 32-bit instruction word, Field Extractor → Decoder
    Inactive wire
    Path unused this cycle (e.g. memory write during an ALU instruction)
    Why does the PC start at 0x80000000?

    The RISC-V privileged spec reserves 0x00000000–0x7FFFFFFF for memory-mapped I/O and boot ROM. Physical DRAM begins at 0x80000000 in virtually all RISC-V SoC designs — so the PC resets there.

    The official riscv-tests compliance suite links every test binary to this address. As a signed 32-bit integer it looks like −2,147,483,648, which is why the HEX toggle exists.

    RV32I Instruction Formats
    Rfunct7 | rs2 | rs1 | funct3 | rd | opcodeADD, SUB, AND…
    Iimm[11:0] | rs1 | funct3 | rd | opcodeADDI, LW, JALR…
    Simm[11:5] | rs2 | rs1 | funct3 | imm[4:0] | opcodeSW, SH, SB
    Bimm[12,10:5] | rs2 | rs1 | funct3 | imm[4:1,11] | opcodeBEQ, BNE, BLT…
    Uimm[31:12] | rd | opcodeLUI, AUIPC
    Jimm[20,10:1,11,19:12] | rd | opcodeJAL
    Component Guide
    PC
    Program Counter — holds the address of the current instruction. Updated every cycle to PC+4 or a branch/jump target.
    PC+4
    PC+4 Adder — always computes the next sequential address. Also written to rd for JAL/JALR as the return address.
    PC/RS1+Imm
    Branch Target Adder — computes PC+imm for branches/JAL or rs1+imm for JALR (selected by MUX T).
    MUX
    Next-PC MUX — selects between PC+4 and the branch target. Driven by pc_src from Branch Logic.
    IMEM
    Instruction Memory — read-only. The PC is the address; the 32-bit instruction word is output combinationally each cycle.
    FE
    Field Extractor — splits the instruction into opcode[6:0], rd[11:7], funct3[14:12], rs1[19:15], rs2[24:20], funct7[31:25].
    IMM GEN
    Immediate Generator — sign-extends the immediate field. Handles all six RV32I formats (R/I/S/B/U/J) based on opcode.
    RF
    Register File — 32 × 32-bit registers (x0–x31). x0 is hardwired to zero. Two async reads (rs1, rs2), one sync write (rd).
    MUX A/B
    ALU Source MUXes — MUX A picks rs1 or PC (AUIPC). MUX B picks rs2 or sign-extended immediate (I/S/B/U/J-type).
    ALU
    Arithmetic Logic Unit — ADD, SUB, AND, OR, XOR, SLL, SRL, SRA, SLT, SLTU, PASS_B (LUI). Outputs a zero flag for branches.
    Branch
    Branch Logic — evaluates the branch condition using funct3 and the ALU result. Outputs pc_src to the Next-PC MUX.
    DMEM
    Data Memory — byte-addressable read/write memory. Supports byte, halfword, and word accesses (LB/LH/LW, SB/SH/SW) via funct3.
    WB MUX
    Writeback MUX — selects the value written to rd: ALU result, memory read data (loads), or PC+4 (JAL/JALR).
    Decoder
    Decoder / Control Unit — purely combinational. Decodes opcode+funct3+funct7 into all control signals: reg_write_en, alu_op, alu_src_mux_1/2, dmem_wr_en, dmem_rd_en, writeback_mux, branch, jump, PC_or_Rs1_mux.
