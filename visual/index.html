    What is this?

    An interactive, cycle-accurate visualizer for a RV32I single-cycle CPU built in SystemVerilog. Write RISC-V assembly, assemble it, then step through each cycle to watch data and control signals flow through every wire in the datapath in real time.

    How to Use
    1Write assembly in the editor or pick an example from the dropdown.
    2Click Assemble & Load to compile and reset the CPU.
    3Use ▶| to step one cycle, ▶ to play continuously, ◀◀ to step back.
    4Toggle HEX in the toolbar for readable addresses (PC starts at 0x80000000 — see below).
    5Pan by dragging the canvas, zoom with scroll or the +/− buttons.
    Wire Color Legend
    Data wire
    32-bit values — PC, register data, ALU result, memory read data
    Control wire (dashed)
    1-bit signals from the Decoder — mux selects, write-enables, ALU op
    Instruction bus
    Raw 32-bit instruction word, Field Extractor → Decoder
    Inactive wire
    Path unused this cycle (e.g. memory write during an ALU instruction)
    Why does the PC start at 0x80000000?

    The RISC-V privileged spec reserves 0x00000000–0x7FFFFFFF for memory-mapped I/O and boot ROM. Physical DRAM begins at 0x80000000 in virtually all RISC-V SoC designs — so the PC resets there.

    The official riscv-tests compliance suite links every test binary to this address. As a signed 32-bit integer it looks like −2,147,483,648, which is why the HEX toggle exists.

    RV32I Instruction Formats
    Rfunct7 | rs2 | rs1 | funct3 | rd | opcodeADD, SUB, AND…
    Iimm[11:0] | rs1 | funct3 | rd | opcodeADDI, LW, JALR…
    Simm[11:5] | rs2 | rs1 | funct3 | imm[4:0] | opcodeSW, SH, SB
    Bimm[12,10:5] | rs2 | rs1 | funct3 | imm[4:1,11] | opcodeBEQ, BNE, BLT…
    Uimm[31:12] | rd | opcodeLUI, AUIPC
    Jimm[20,10:1,11,19:12] | rd | opcodeJAL
    Component Guide
    PC
    Program Counter — holds the address of the current instruction. Updated every cycle to PC+4 or a branch/jump target.
    PC+4
    PC+4 Adder — always computes the next sequential address. Also written to rd for JAL/JALR as the return address.
    PC/RS1+Imm
    Branch Target Adder — computes PC+imm for branches/JAL or rs1+imm for JALR (selected by MUX T).
    MUX
    Next-PC MUX — selects between PC+4 and the branch target. Driven by pc_src from Branch Logic.
    IMEM
    Instruction Memory — read-only. The PC is the address; the 32-bit instruction word is output combinationally each cycle.
    FE
    Field Extractor — splits the instruction into opcode[6:0], rd[11:7], funct3[14:12], rs1[19:15], rs2[24:20], funct7[31:25].
    IMM GEN
    Immediate Generator — sign-extends the immediate field. Handles all six RV32I formats (R/I/S/B/U/J) based on opcode.
    RF
    Register File — 32 × 32-bit registers (x0–x31). x0 is hardwired to zero. Two async reads (rs1, rs2), one sync write (rd).
    MUX A/B
    ALU Source MUXes — MUX A picks rs1 or PC (AUIPC). MUX B picks rs2 or sign-extended immediate (I/S/B/U/J-type).
    ALU
    Arithmetic Logic Unit — ADD, SUB, AND, OR, XOR, SLL, SRL, SRA, SLT, SLTU, PASS_B (LUI). Outputs a zero flag for branches.
    Branch
    Branch Logic — evaluates the branch condition using funct3 and the ALU result. Outputs pc_src to the Next-PC MUX.
    DMEM
    Data Memory — byte-addressable read/write memory. Supports byte, halfword, and word accesses (LB/LH/LW, SB/SH/SW) via funct3.
    WB MUX
    Writeback MUX — selects the value written to rd: ALU result, memory read data (loads), or PC+4 (JAL/JALR).
    Decoder
    Decoder / Control Unit — purely combinational. Decodes opcode+funct3+funct7 into all control signals: reg_write_en, alu_op, alu_src_mux_1/2, dmem_wr_en, dmem_rd_en, writeback_mux, branch, jump, PC_or_Rs1_mux.
    RISC‑V Pipeline Visualizer — About
    View RTL on GitHub
    What is this?

    An interactive, cycle-accurate visualizer for a RV32I 5-stage pipelined CPU built in SystemVerilog. Up to five instructions execute simultaneously — one per stage. Watch pipeline registers latch values, forwarding paths activate, and stall/flush bubbles propagate in real time.

    How to Use
    1Write assembly or pick a pipeline example (Forwarding, Load-Use Stall, Branch Flush).
    2Click Assemble & Load to compile and reset the CPU.
    3Use ▶| to step one cycle — each step advances all five stages simultaneously.
    4Watch the stage chips in the header: they show which instruction is in each stage this cycle.
    5Green forwarding wires light up when data is bypassed; pipeline registers turn red on flush / dim on stall.
    6Step back with ◀◀ to replay any previous cycle (up to 500 cycles of history).
    Wire Color Legend
    Data wire
    32-bit values — PC, register data, ALU result, memory read data
    Control wire (dashed)
    1-bit signals — mux selects, write-enables, branch/jump, ALU op
    Forwarding wire
    Bypass paths that carry a result from a later pipeline stage back to the EX stage inputs
    Inactive wire
    Path unused this cycle
    The 5 Pipeline Stages
    IF
    Instruction Fetch — PC drives IMEM; output latched into IF/ID pipeline register. PC updates to PC+4 or branch target.
    ID
    Instruction Decode — Field Extractor splits bits, Decoder produces control signals, Register File reads rs1/rs2, Imm Gen sign-extends the immediate. Everything latches into ID/EX.
    EX
    Execute — Forwarding muxes select the freshest rs1/rs2 values, ALU input muxes add the immediate if needed, ALU computes the result. Branch Logic decides pc_src. Results latch into EX/MEM.
    MEM
    Memory — Data Memory is read (load) or written (store). All other instructions pass through. Results latch into MEM/WB.
    WB
    Writeback — WB MUX selects between ALU result, memory read data, or PC+4. The chosen value is written back to the Register File.
    Hazard Handling
    FWD
    Data Forwarding (EX stage) — The Forwarding Unit detects when an instruction in EX needs a value that is still in EX/MEM or MEM/WB (not yet written back). It steers the forwarding muxes (FA, FB) to bypass the pipeline register directly. Eliminates most RAW hazards with zero penalty.
    HAZ
    Load-Use Stall (Hazard Unit) — A load followed immediately by an instruction that uses the loaded value cannot be forwarded (the data isn't ready until end of MEM). The Hazard Unit inserts one bubble: it stalls IF and ID by freezing their pipeline registers and injects a NOP into EX. The pipeline register turns dim to show the stall.
    BR
    Branch Flush (always not-taken) — Branches are resolved in EX. The CPU always fetches PC+4 speculatively. If the branch is taken, the two instructions already in IF and ID are wrong — the Hazard Unit flushes them by writing NOP bubbles into IF/ID and ID/EX. Pipeline registers turn red to show the flush. Cost: 2 cycles per taken branch.
    EX Stage Component Guide
    FA / FB
    Forwarding MUXes — Select the freshest value for rs1 (FA) and rs2 (FB): 00 = register file, 01 = EX/MEM bypass, 10 = MEM/WB bypass. Controlled by the Forwarding Unit.
    MUX A/B
    ALU Source MUXes — MUX A picks rs1 or PC (AUIPC). MUX B picks rs2 or sign-extended immediate. Selected by alu_src_mux_1/2 control signals from ID/EX.
    ALU
    Arithmetic Logic Unit — ADD, SUB, AND, OR, XOR, SLL, SRL, SRA, SLT, SLTU, PASS_B (LUI). The alu_op select comes from ID/EX Ctrl, entering the ALU from below.
    PC+I / PC+4
    Branch Target Adders — PC+Imm computes the branch/JAL target. PC+4 computes the return address for JAL/JALR. Both feed the Branch MUX.
    MUX
    Branch MUX (BR_MUX) — Selects between PC+Imm (branches, JAL) and rs1+Imm (JALR). The select bit (PC_or_Rs1_mux) comes from ID/EX Ctrl.
    Branch
    Branch Logic — Takes alu_result, funct3, branch, jump signals. Outputs pc_src: 1 = take branch/jump, 0 = PC+4. pc_src feeds back to the IF stage NEXT_MUX.
    FWD Unit
    Forwarding Unit — Compares rs1/rs2 addresses from ID/EX against rd addresses in EX/MEM and MEM/WB. Outputs forward_a and forward_b 2-bit selects for FA/FB.
    HAZ Unit
    Hazard Detection Unit — Monitors ID/EX mem_read and rs1/rs2 addresses from IF/ID. Asserts stall when a load-use hazard is detected; triggers flush on taken branches.
